Why I choose to develop programs using a new method

I would normally learn all the basics from the usual resources: videos and articles. Then I would do coding exercises from different platforms and websites. The thing about this is that this method does not allow me to apply what I have learnt into a technical/complex program. I also ended up doing projects that are too easy (Fibonacci sequence or guess the number) and I don’t grow as a developer because I focus on completion, testing, documentation. These things are not bad but I am not going to get better if I just focus on those things.

So, I decided to go for another method.

I have decided to complete intermediate projects that others had already done first. Then do an advanced project.  I would have an easier time and have the tools necessary to do an advanced project without it being so daunting because I would not have to start from scratch.

Aim

My aim is to make an AI that uses my computer as its resource. I would be able to ask question about what does “document.pdf” say in 2 paragraphs or check my outlook and see if I have new emails on a specific topic.

Second project

Explain the code: An AI agent that is able to search the internet. It uses agents to:

  • Make a decision if the user prompt should be answered using the internet.
  • Makes a search query.
  • Analyses the data scraped.
  • Another that selects the best search results.
  • Generate a response to the user.

This is the link: https://www.youtube.com/watch?v=9KKnNh89AGU

What I learnt:

  • Each agent has its own template and they all work in the same way.
  • I thought that there could only be one agent per file since the would have to work concurrently.
  • I learnt about retrieving data from a search result compared to a website
  • I learnt about streams - you can wait for the full response or use the chunks individually
  • There is another way to make agents without a template

Things I thought about to improve the code:

  • I could compress the context once it gets to a certain size but I don't know if it'll be relevant cause it would need to add more functions to decrypt it.
  • I could measure the big O of the program and find where the bottlenecks are.
  • I realised that I needed more understanding in chaining.
  • Issues that I found with the model I was using until now (llama3:latest):
  • It takes a lot of space. It cannot run with a lot of other programs running in the background
  • For now, I will keep using llama3:latest because I don’t what to break the first program and not use more space in the RAM for another one.

Third Project

It is an AI agent that looks at a database filled with data from a document for the answers.

This is the link: https://www.youtube.com/watch?v=E4l91XKQSgw

I noticed that this project doesn't have context/history. Which I thought was strange since it would not be able to customise the response for the user but once I thought about it, I think it makes sense because the purpose it retrieves information from the document.

I learnt about

  1. How to add the results from the vector into the model.
  2. A RAG with a foundational definition for RAG.
  3. How to use a vector database.
  4. How to use a document.
  5. Basics on getting information from a csv file.
  6. How to not always regenerate something and find if it exists.
  7. What is a document.
  8. That you don't always need history.

I am guessing that depending on the file there is a module to retrieve information from it and do things with that data so file.open() is not relevant, maybe?

Research for preparation for the merge project

I decided to some research for the final project of part 1 because I did not understand everything in the programs I had copied. I knew I needed a good base understanding for when I would have to make the next project on my own.

Things I learnt:

  • I learnt more about chaining. How does it work exactly and if there are other ways to chain templates with models.
  • I also learnt about the attributes in Document and Chroma classes.
  • Why there is a need for a document and a vector store when the file is already in the computer.

I looked at videos on different AI topics that are focused on llama

Things I learnt from the videos:

  • There are different types of complex templates depending on the model and each has its own syntax.
  • What is a model file and how to change it
  • What is context size and how to change it.
  • The RAG process.
  • What is the difference between a template and a model file.
  • There are different embedding modules.
  • The addition of data is done in a way that adheres to the format to vector store.

At this point I felt scared but prepared for the task ahead.
